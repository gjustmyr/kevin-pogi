MIGRATION INSTRUCTIONS: "Cleared" to "Validated" Status Change
================================================================================

OVERVIEW:
---------
Changed requirement submission status from "cleared" to "validated" to eliminate
confusion between:
  - Individual files being validated/checked by dean
  - Faculty overall clearance status

TERMINOLOGY NOW:
  ✓ Files: "validated" (approved/checked by dean)
  ✓ Faculty: "cleared" (overall status when all requirements done)

DEPLOYMENT STEPS:
-----------------

1. BACKUP YOUR DATABASE FIRST!
   ```bash
   mysqldump -u root -p commision_db > backup_before_migration.sql
   ```

2. Stop the application:
   ```bash
   # Stop backend server
   # Stop frontend dev server
   ```

3. Run the database migration:
   ```bash
   cd backend
   mysql -u root -p commision_db < migrations/2026-03-06-rename-cleared-to-validated.sql
   ```

4. Verify the migration:
   ```sql
   USE commision_db;
   SELECT status, COUNT(*) as count 
   FROM requirement_submissions 
   GROUP BY status;
   -- Should show: pending, validated, returned (NO "cleared")
   ```

5. Install backend dependencies (if needed):
   ```bash
   cd backend
   npm install
   ```

6. Install frontend dependencies (if needed):
   ```bash
   cd client
   npm install
   ```

7. Start the application:
   ```bash
   # Terminal 1: Start backend
   cd backend
   npm start

   # Terminal 2: Start frontend
   cd client
   npm start
   ```

8. TEST THE CHANGES:
   - Login as Dean
   - Go to Accomplishments Monitoring
   - Select a faculty with submitted requirements
   - Click "Validate" button (was "Clear")
   - Check statistics show "X Validated" (was "X Cleared")
   - Verify faculty badge updates correctly
